How does JavaScript works?

Everything in javaScript happens inside execution context. It has two components inside of it. The first component is called memory component or variable environment. This is the place where all the variable and functions are stored as key value pairs. The second component is code component or thread of execution. This is the place where all the codes are executed one line at a time. So when a javascript program is run at first a global execution context is created and pushed into call stack inside of javascript engine. Then the first phase is started where all the variable takes space from memory, but they are undefined and all the functions are stored in memory. Next the second phase is started where line by line code is executed means all the variable gets their value and for every function execution another local execution context is created and again pushed into call stack. Then the same process start again until the callstack becomes empty.

What does JavaScript Event Loop do?

There is call stack for execution of javascript code. There are also callback queue and microtask queue inside browser for queuing callback functions, time consuming functions or promises related callback functions. Promise callback functions goes into microtask queue and other callback functions like setTimeout, addEventListener,etc goes to callback queue. But to execute these callback functions they need to be pushed into call stack. Event loop works here as a gate keeper means it always check whether the call stack empty or not. If it's empty then it check if there any callback functions inside callback queue or microtask queue. If there is then the event loop pop from the callback queue or microtask queue and pushed into call stack. But microtask queue has higher priority.So Event loop check it first.
